# 動作環境
＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝

TextRunRun は Unicode版のAutoHotkey上で動作します。  
AutoHotkey_L  Unicode版  の事をいいます。

OS は、Windowsでのみ使用できます。  
Windows XP / 2003 / Vista / 7 / 8 / 10 (32-bit、64-bit両対応)

AutoHotkey_L が動作する環境であれば可能だと思います。

Windows Me や  
Windows 2000 でも可能だと思います。


-----------------------------------------------------

※ AutoHotkey の派生版である 「AutoHotkey_L Unicode版」でのみ動作します。

元の AutoHotkey では使えません。


それと  AutoHotkey 1.1.xx のバージョンのみで使えます。  
AutoHotkey 2.xx のバージョンでは使えません。

# 主な特徴
＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝

このソフトの「入力欄のあるウインドウ」を　「テキスト」と名付けています。  
その「テキスト」に処理内容を記入して Enter で実行するというのが基本的な使い方です。


「テキスト」に指定して実行できる事は主に以下の事があります。

  - ファイルパスを記入して実行

  - 「登録していたものの名前」を記入して実行

  - AutoHotkey のスクリプトを記入して実行

　AutoHotkey のスクリプトを実行できる事が大きな特徴です。  
　これにより、幅広い処理が指定できます。



＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝
# ファイルの構成
＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝

TextRunRun.exe				本体です。  

trrEvery（フォルダ）		TextRunRun に必要なファイルやフォルダは  
							この中に構成されるようになっています。

trrEvery  フォルダ内  

	trrEvery\user			ユーザーが設定した事はこのフォルダ内に反映されます。

	trrEvery\doc			画像を置いておく場所です。  
							決められたファイル名の画像を置くと  
							ソフトのウインドウ内に反映されます。

	trrEvery\fromFileName		「使い方.txt」のファイル内で説明しますが、  
								ユーザーがファイルを作ることで  
								アイテムやショートカットキーを  
								作ることが可能になる場所です。


これらのフォルダー内がカラの場合、
TextRunRun.exe を実行すると、自動的に作成するようになっています。
ただし、trrEvery\doc の画像などは自動的に作成できません。

# インストール
＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝

現時点では、インストーラーはありません。

○レジストリはいじりません  
　USBなどのストレージから使用することも可能です。


ダウンロードしたファイルを解凍ソフトで展開してください。

その後に好きなフォルダ内に配置して使ってください。  
TextRunRun.exe  を実行すると、  
TextRunRun に必要なファイルやフォルダが同じフォルダ内に自動的に作成されます。

----------------------------------------------------

Program Files  などの UAC管理下のフォルダの中では  
このソフトは使用できません。

それ以外のフォルダに配置して使ってください。

----------------------------------------------------

# アンインストール
＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝

レジストリはいじっていません。  
ユーザーが配置した TextRunRun のフォルダを削除すればＯＫです。

TextRunRun が自動的に作るものは、
trrEvery  のフォルダ内にのみ作られます。  
ですが、ユーザーが故意に作ったものは、ユーザーが覚えておく必要があります。

＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝
# バージョンアップ
＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝

1. TextRunRun を必ず終了させておいてください。

2. 解凍したばかりならばそのままフォルダ内のファイルを
   すべて上書きすればOKです。

-------------------------------------------------

「trrEvery\user」のフォルダをバックアップしておく事を薦めます。

-------------------------------------------------

　もし、TextRunRun を実行したまま上書きしてしまった場合、
　設定用のファイルなどが、作者が想定しない形で上書きなどをするかもしれません。

　それによるトラブルが発生するかもしれないので、
　バージョンアップする際は、念のため  
　「trrEvery\user」  
　のフォルダをバックアップしておく事を薦めます。

＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝
# バックアップ
＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝

	trrEvery\user  
	のフォルダと  
	trrEvery\exceptionOfUser.ini  
	のファイルをどこかに保存しておいてください。

	ユーザーが設定した内容は以上のファイル内に設定内容が保存されます。


	trrEvery\fromFileName  
	のフォルダ内のファイルも設定に関係するファイルですが、  
	これはユーザーが意図的に作るものです。

	このフォルダ内のファイルは、TextRunRun の内部処理が操作をする事はありません。  
	バックアップするかどうかは、ユーザーが任意で決めてください。

# キー配置変換ソフトとの相性
＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝

TextRunRun は、AutoHotkey と同じ様に
組み合わせキーといって、変換キーや無変換キーを Ctrl キーの様に使用したりする事が可能です。

[例]
  - vk1C & a   で   変換 + a

  - vk1D & a   で   無変換 + a


ただし、これを使った場合は「キー配置変換ソフト」との相性はあまりよくありません。

作者のおすすめとしては、
キー配置変換ソフトの方をベースにして、
そちらで変換キーや無変換キーなどを便利に使える様にしておき、
TextRunRun では、組み合わせキーは使用しない方がいいと思います。  
（便利ですが、TextRunRun は不完全なところが多いからです。）

基本、変換キーや無変換キーなどの組み合わせキーの使用は  
自分独自の AutoHotkey を別に起動して使用し、そちらで使う方が良いと思います。

fromFileName は完全にユーザーが作るものなので、それが原因の不具合は対処できません。
